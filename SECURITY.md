# Security Policy

Thank you for helping keep **Remem** secure.

## Supported Versions

Only the latest stable release is actively supported with security updates.

| Version        | Supported |
| -------------- | --------- |
| Latest stable  | ✅         |
| Older releases | ❌         |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not create a public GitHub issue**.

Instead, report it privately by emailing:

**[harshvardhan1483@gmail.com](mailto:harshvardhan1483@gmail.com)**

Please include:

* A description of the issue
* Steps to reproduce
* Potential impact
* Suggested mitigation (if available)

You can also include a proof of concept if appropriate.

## Response Process

I will make every effort to:

* Acknowledge the report within 72 hours.
* Investigate the issue promptly.
* Keep you informed throughout the resolution process.
* Release a fix as soon as practical.
* Publicly acknowledge responsible disclosure (if you wish).

## Scope

Examples include, but are not limited to:

* Remote code execution
* Dependency vulnerabilities
* Arbitrary file access
* Authentication or authorization issues
* Data leakage
* Unsafe deserialization
* Cache poisoning
* Prompt injection that compromises security guarantees
* Denial-of-service vulnerabilities

## Disclosure Policy

Please allow reasonable time for a fix before publicly disclosing any vulnerability.

Responsible disclosure helps protect users while the issue is being resolved.

Thank you for helping improve the security of Remem.
